### The agent loop (from session/processor.ts)

The processor listens to LLM streaming events and orchestrates tool calls:
1. LLM stream emits `tool-call` → processor creates a `ToolPart` in the
   session, calls `permission.ask()` if the tool requires it, then invokes
   `tool.execute(args, ctx)`.
2. Tool returns `ExecuteResult` → processor updates the `ToolPart` to
   `completed` with output.
3. AI SDK feeds tool results back into the LLM stream automatically.
4. Loop until LLM emits a final text response with no more tool calls.

### Permission engine

Wildcards patterns like `shell.*`, `write.*`, `read.**`. Rulesets have
three actions: `allow`, `deny`, `ask`. When `ask`, the engine emits a
`Permission.Event.Asked` bus event, a UI subscriber shows the request,
user replies, engine resolves the pending `Deferred`. Approval decisions
can be persisted to `state.approved` so future asks of the same pattern
auto-allow within the session.

### Session runtime (the agent loop)

```ts
async function runAgent(input: {
  agent: AgentDef;
  prompt: string;
  history?: Message[];
  ctx: ToolContext;
  onEvent(evt: RuntimeEvent): void;
}) {
  const tools = registry.forAgent(input.agent);
  const messages = [
    { role: "system", content: input.agent.systemPrompt },
    ...input.history ?? [],
    { role: "user", content: input.prompt },
  ];
  for (let i = 0; i < MAX_ITER; i++) {
    input.onEvent({ type: "iteration", n: i });
    const stream = await llm.streamChat({
      model: input.agent.model,
      messages,
      tools: tools.map(toolToSchema),
    });
    let assistantText = "";
    const pendingCalls: ToolCall[] = [];
    for await (const chunk of stream) {
      if (chunk.type === "text") { assistantText += chunk.text; input.onEvent(chunk); }
      if (chunk.type === "tool_call") { pendingCalls.push(chunk); input.onEvent(chunk); }
    }
    messages.push({ role: "assistant", content: assistantText, tool_calls: pendingCalls });
    if (pendingCalls.length === 0) break;
    for (const call of pendingCalls) {
      const def = registry.get(call.name);
      const result = await runTool(def, call, input.ctx, input.onEvent);
      messages.push({ role: "tool", tool_call_id: call.id, content: result.output });
    }
  }
  return { messages, done: true };
}

async function runTool(
  def: ToolDef, call: ToolCall, ctx: ToolContext, emit: (e: RuntimeEvent) => void
): Promise<ToolResult> {
  emit({ type: "tool_start", callID: call.id, tool: def.id, args: call.args });
  // Permission gate
  const rule = evaluatePermission(def.tier, agent.permission);
  if (rule === "deny") {
    return errorResult(def, "Denied by policy");
  }
  if (rule === "ask") {
    const { approved } = await ctx.ask({
      tool: def.id,
      summary: renderToolSummary(def, call.args),
    });
    if (!approved) return errorResult(def, "User rejected");
  }
  // Parse + execute
  const parsed = def.parameters.safeParse(call.args);
  if (!parsed.success) {
    return errorResult(def, `Invalid arguments: ${parsed.error.message}`);
  }
  try {
    const result = await def.execute(parsed.data, ctx);
    emit({ type: "tool_done", callID: call.id, result });
    return result;
  } catch (e) {
    const result = errorResult(def, String(e));
    emit({ type: "tool_error", callID: call.id, error: String(e) });
    return result;
  }
}
```

### Permission (simple version)

```ts
export type Action = "allow" | "ask" | "deny";
export interface PermissionRule {
  tier: ToolTier;
  toolId?: string;                // optional: rule for specific tool
  action: Action;
}
export function evaluatePermission(tier: ToolTier, rules: PermissionRule[]): Action {
  const specific = rules.find((r) => r.tier === tier && r.toolId);
  const generic = rules.find((r) => r.tier === tier && !r.toolId);
  return specific?.action ?? generic?.action ?? "deny";
}
```

For MVP, `ask` triggers an approval dialog. Session-scoped approvals
cached in the runtime state so the user isn't asked twice for the same
pattern.
